# Multi-Agent Requirement Workflow MVP

一个基于多 Agent 协作的智能需求分析与任务拆解 MVP 系统。

本项目通过多个 AI Agent 的分工协作，将自然语言产品需求自动转化为结构化的需求分析、开发任务、风险评估和项目执行方案，帮助研发团队提升需求评审效率、降低沟通成本，并更早识别项目风险。

---

## 项目背景

在传统研发流程中，产品经理提出一个自然语言需求后，通常需要研发负责人、前端、后端、测试、架构师等多方反复沟通，才能完成：

- 需求澄清
- 功能拆解
- 技术方案评估
- 测试范围确认
- 项目风险识别
- 排期和执行计划制定

这个过程往往存在以下问题：

1. **需求理解成本高**  
   自然语言需求容易存在模糊、不完整或隐含信息，需要多人多轮沟通才能明确。

2. **任务拆解依赖人工经验**  
   不同负责人拆解任务的粒度、优先级和完整度可能不一致，容易遗漏关键工作。

3. **风险识别滞后**  
   技术风险、交付风险、依赖风险通常在开发过程中才暴露，导致返工或延期。

4. **跨角色沟通成本高**  
   产品、研发、测试、架构之间需要反复对齐，影响项目启动效率。

本项目希望通过多 Agent 协作，将这些流程自动化、结构化、标准化。

---

## 项目解决的核心痛点

本项目主要解决的是：

> 如何将一个模糊的自然语言产品需求，快速转化为一份结构化、可执行、可评估风险的研发执行方案。

系统通过 AI Agent 自动完成需求理解、任务拆解、风险评估和方案汇总，帮助团队减少人工评审成本，提高项目启动效率。

核心价值包括：

- 将自然语言需求自动转化为结构化需求分析
- 自动拆解前端、后端、数据库、测试等研发任务
- 自动识别技术风险、产品风险和交付风险
- 自动生成项目执行方案
- 支持 Web 页面和 CLI 命令行两种运行方式
- 支持 Mock 模式，便于无 API Key 快速演示
- 支持接入真实 OpenAI API，用于真实场景测试

## 核心逻辑流

本项目采用 **长链推理 + 多 Agent 协作** 的工作流设计。

整体流程如下：

```text
自然语言需求输入
        ↓
需求理解 Agent
        ↓
任务拆解 Agent
        ↓
风险评估 Agent
        ↓
汇总决策 Agent
        ↓
最终项目执行方案输出
